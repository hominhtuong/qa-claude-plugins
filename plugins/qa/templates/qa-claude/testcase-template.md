## Language
- Default: **Vietnamese with diacritics** for all content (description, precondition, steps, expected, section titles). "Đăng nhập" NOT "Dang nhap". Switch with `language: English` in the request.
- Sanitize AI artifacts: `—` → `-`, `→` → `=>`.

## Sheet header (rows 1-7, per sheet)
- Row 1: `Link DOC:` | (link) … `Passed` | `=COUNTIF(F<data>, D1)`
- Row 2: `Link Figma:` | (link) … `Failed` | `=COUNTIF(F<data>, D2)`
- Row 3: `Create by` | **<your team / author name>** … `Not start` | `=COUNTIF(...)`
- Row 4: `Create Date` | {YYYY-MM-DD} … `Cancel` | `=COUNTIF(...)`
- Row 5: `Testcases` | `=SUM(E1:E4)`
- Row 6: `Time Est (1 round):` | auto-formula (sum of column I → giờ + phút)
- Row 7: `Story Point:` | {SP} ({role}) — optional, omit if your team doesn't use it

## Conventions
- **TC_ID resets per sheet** (each module sheet starts at `TC_001`). Multi-module → one sheet per module, each with its own header + COUNTIF.
- **Status options** (column F/G dropdown): `PASSED`, `FAILED`, `NOT START`, `CANCEL`; default `NOT START`. Colors: Passed `#D9F5D6`, Failed `#FBBFBC`, Not start `#E1EAFF`, Cancel `#DEE0E3`.
- **Multi-result TC**: one TC description with several expected results → each result on its own row; merge column B + C across those rows (`merge_with_prev` from the 2nd row).
- **Freeze** the top 8 rows; header rows 7-8 bold/centered, bg `#CCFFFF`; section rows bg `#ECE2FE`; font Arial; thin border `#D0D0D0`.

## Writing quality (summary — full rules in the plugin's `test-quality.md`)
- 1 objective per TC; steps are actions only; expected results separated out.
- **Mandatory negative coverage** per section (a section with 0 negative cases is a red flag).
- No redundant expected, no pure color-code checks.
- Steps must be reproducible by any QC without extra explanation.

## Upload destination (priority chain)
Local `.xlsx` in `results/<feature>/` is always saved (hard backup). Then upload once (after merging phases) via: Lark Drive (`LARK_DRIVE_FOLDER_ID`) → Google Sheets (`GOOGLE_DRIVE_FOLDER_ID`) → local-only. Folder IDs come from `.claude/qa-claude/.plugin.env` (not committed).
